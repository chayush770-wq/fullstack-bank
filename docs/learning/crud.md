# CRUD Principle

## מה זה CRUD?

CRUD הוא עיקרון שמתאר את ארבע הפעולות הבסיסיות שניתן לבצע על מידע במערכת.

כל מערכת שעובדת עם נתונים משתמשת בפעולות האלה.

---

## ארבע הפעולות

### Create (יצירה)
יצירת מידע חדש במערכת

דוגמאות:
- יצירת משתמש
- יצירת פעולה כספית (transaction)

---

### Read (קריאה)
קבלת מידע מהמערכת

דוגמאות:
- צפייה ביתרה
- צפייה בהיסטוריית פעולות

---

### Update (עדכון)
שינוי מידע קיים

דוגמאות:
- שינוי סיסמה
- עדכון פרטי משתמש

---

### Delete (מחיקה)
מחיקת מידע מהמערכת

דוגמאות:
- מחיקת משתמש
- מחיקת נתונים

---

## חיבור ל־HTTP

ב־REST API נהוג למפות את CRUD לשיטות HTTP:

- Create → POST  
- Read → GET  
- Update → PUT / PATCH  
- Delete → DELETE  

---

## דוגמא מהפרויקט

### Transactions

- POST /transactions → Create (יצירת פעולה כספית)  
- GET /transactions → Read (קבלת פעולות)  

---

## הערה חשובה

לא כל פעולה במערכת היא CRUD.

לדוגמא:
- login  
- logout  

אלו פעולות (Actions) ולא פעולות ישירות על מידע.

---

## סיכום

CRUD הוא הבסיס לניהול מידע במערכות,  
ומאפשר לתכנן API בצורה מסודרת וברורה.
